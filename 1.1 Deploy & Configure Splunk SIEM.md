## Splunk Core SIEM

SIEM (Security Information and Event Management) is the backbone of a SOC environment. It enables centralized log collection, normalization, correlation, and alerting across infrastructure components.

**Splunk** is a widely adopted enterprise SIEM platform used globally for security monitoring, threat detection, and incident investigation.

---

# 🎯 Objective

Install, configure, and perform initial hardening of **Splunk Enterprise SIEM**, including forwarder setup and index configuration.

---

# 📌 LAB 1.1 – Deploy & Configure Splunk SIEM

---

# 🛠 STEP 1: Download & Install Splunk Enterprise

Download Splunk Enterprise from:

```text id="spl1"
https://www.splunk.com
```

Navigate to:

* Free Trial → Download Splunk Enterprise (`.deb / .rpm / .tgz`)

---

## Install Splunk (Linux)

```bash id="spl2"
sudo dpkg -i splunk--linux-2.6-amd64.deb
```

---

## Start Splunk & Accept License

```bash id="spl3"
sudo /opt/splunk/bin/splunk start --accept-license
```

---

## Enable Boot Start

```bash id="spl4"
sudo /opt/splunk/bin/splunk enable boot-start
```

---

## Access Web Interface

Open browser:

```text id="spl5"
http://localhost:8000
```

Login:

* Username: `admin`
* Password: *(set during installation)*

---

# ⚙️ STEP 2: Configure Receiving Port (Indexer)

Splunk indexers receive data from forwarders via a dedicated port.

---

## Configure via UI

Navigate:

```text id="spl6"
Settings → Forwarding and Receiving → Configure Receiving → New Receiving Port
```

Set:

* Port: `9997`

> ⚠️ Port 9997 is the default Splunk forwarder-to-indexer communication port.

---

## Configure via CLI

```bash id="spl7"
sudo /opt/splunk/bin/splunk enable listen 9997 -auth admin:password
```

---

# 📡 STEP 3: Install Universal Forwarder (Linux Agent)

The Splunk Universal Forwarder is used to collect and send logs from endpoints to the indexer.

---

## Install Forwarder

```bash id="spl8"
sudo dpkg -i splunkforwarder--linux-2.6-amd64.deb
```

---

## Start Forwarder

```bash id="spl9"
sudo /opt/splunkforwarder/bin/splunk start --accept-license
```

---

## Configure Forwarding Server

```bash id="spl10"
sudo /opt/splunkforwarder/bin/splunk add forward-server <IP>:9997 -auth admin:password
```

---

## Monitor Linux Logs

```bash id="spl11"
sudo /opt/splunkforwarder/bin/splunk add monitor /var/log/syslog
```

---

## Restart Forwarder

```bash id="spl12"
sudo /opt/splunkforwarder/bin/splunk restart
```

---

# 🗂 STEP 4: Configure indexes.conf

Create a custom index for security logs.

---

## Edit Configuration File

```bash id="spl13"
sudo nano /opt/splunk/etc/system/local/indexes.conf
```

---

## Index Configuration

```ini id="spl14"
[security_logs]
homePath = $SPLUNK_DB/security_logs/db
coldPath = $SPLUNK_DB/security_logs/colddb
thawedPath = $SPLUNK_DB/security_logs/thaweddb
```

---

## Restart Splunk

```bash id="spl15"
sudo /opt/splunk/bin/splunk restart
```

# 📘 BEST PRACTICE TIPS

---

## 🔐 Security Hardening

* Change default **admin password immediately**
* Never retain default credentials in production

---

## 👤 Service Account Best Practice

* Run Splunk using a **non-root service account** in production environments

---

## 🔒 Encryption & Transport Security

* Enable **TLS/SSL encryption** for:

  * Port `8089` (management)
  * Port `9997` (forwarders)
* Use organization-approved **CA certificates**

---

## 📊 Data Retention & Compliance

* Configure index retention policies:

  * Example: `365 days` retention as per compliance requirements

---

## ⚙️ Enterprise Scaling

* Use **Deployment Server (DS)** to centrally manage:

  * Forwarder configurations
  * App deployments
  * Policy updates
