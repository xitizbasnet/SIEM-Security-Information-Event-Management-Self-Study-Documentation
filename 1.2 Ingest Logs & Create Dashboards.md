## 🎯 Objective

Configure Splunk data inputs from multiple sources and build a **SOC security dashboard** for real-time monitoring and threat visibility.

---

# 📌 STEP 1: Add Windows Event Log Input (Universal Forwarder)

Windows event logs are a critical telemetry source for detecting authentication, system, and application-level security events.

---

## 🖥️ Configure via Windows Splunk Forwarder

Navigate:

```text id="win1"
Settings → Data Inputs → Windows Event Log
```

Select the following event channels:

* Security
* System
* Application

---

## ⚙️ inputs.conf Configuration

```ini id="win2"
[WinEventLog://Security]
disabled = false
index = security_logs
```

---

# 📌 STEP 2: Add Syslog Input (Linux / Network Devices)

Syslog is used to ingest logs from Linux servers, firewalls, routers, and network appliances.

---

## 🌐 Configure Splunk Syslog Input

Navigate:

```text id="sys1"
Settings → Data Inputs → UDP
```

Set parameters:

* Port: `514`
* Source Type: `syslog`
* Index: `security_logs`

---

## 🔌 Network Device Configuration

Configure syslog forwarding on network devices:

* Syslog Server IP: `<Splunk Server IP>`
* Port: `514`
* Facility: `local7`

---

## 📊 Validate Syslog Ingestion

```bash id="sys2"
sudo tail -f /opt/splunk/var/log/splunk/splunkd.log | grep syslog
```

---

# 📌 STEP 3: Verify Data Ingestion

Ensure logs are successfully indexed in Splunk.

---

## 🔍 Basic Verification Search

```spl id="vr1"
index=security_logs | head 20
```

---

## 📊 Validate Data Sources

```spl id="vr2"
index=security_logs | stats count by sourcetype
```

---

## ⚠️ Troubleshooting Checklist

If no data appears:

* Verify forwarder configuration (`outputs.conf`)
* Check firewall rules for port `9997`
* Ensure syslog port `514` is open
* Validate index name consistency (`security_logs`)

* Bar Chart

# ⚠️ BEST PRACTICE GUIDELINES

---

## ⚡ Performance Optimization

* Use `tstats` for large datasets (accelerated searches)
* Avoid full-time range searches in dashboards

---

## ⏱️ Time Management

* Always configure a **time-range picker**
* Prevent accidental “All Time” queries in production

---

## 🌍 Data Standardization

* Normalize timestamps to **UTC at ingestion**
* Ensures accurate cross-system correlation

---

## 🏷️ Data Tagging Strategy

Enhance searchability by tagging:

* Environment: `prod / dev`
* Business unit: `finance / IT / security`

Use `props.conf` for structured metadata enrichment
